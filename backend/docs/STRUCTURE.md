# Структура проекта FloodSite

## Обзор структуры

Проект организован по принципу группировки файлов по типам и назначению.

## Структура директорий

```
backend/
├── uploads/              # Загруженные пользователями файлы
│   ├── avatars/         # Аватары пользователей (.jpg, .png, .gif, .webp)
│   ├── projects/        # HEC-RAS проекты (.db файлы)
│   └── files/          # Другие загруженные файлы
│
├── data/                # Данные приложения
│   ├── databases/      # Файлы баз данных (SQLite)
│   └── mbtiles/        # Тайлы карт (.mbtiles)
│
├── docs/                # Документация проекта
│   ├── EMAIL_SETUP.md
│   ├── MIGRATION_README.md
│   ├── SMTP_TESTING.md
│   └── STRUCTURE.md (этот файл)
│
├── scripts/             # Скрипты миграции и утилиты
│   ├── migrate_email_change_fields.py
│   ├── migrate_hecras_metadata.py
│   └── run_migration.py
│
├── migrations/          # Alembic миграции базы данных
│   ├── env.py
│   └── versions/
│
├── database/            # Код базы данных и API
│   ├── models/         # SQLAlchemy модели
│   ├── routers/        # FastAPI роутеры
│   ├── schemas.py      # Pydantic схемы
│   ├── crud.py         # CRUD операции
│   ├── file_paths.py   # Централизованное управление путями
│   └── ...
│
└── main.py             # Точка входа FastAPI приложения
```

## Типы файлов и их расположение

### Загруженные файлы (uploads/)

- **avatars/** - Аватары пользователей
  - Форматы: `.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`
  - Именование: `{user_id}_{random_token}.{ext}`
  - Доступ: через `/api/avatars/{filename}`

- **projects/** - HEC-RAS проекты
  - Формат: `.db` (SQLite базы данных)
  - Именование: `{project_name}.db`
  - Содержит: тайлы карт, метаданные, слои

- **files/** - Другие загруженные файлы
  - Для будущего использования

### Данные приложения (data/)

- **databases/** - Файлы баз данных
  - `floodsitedb.db` - основная база данных приложения (если используется SQLite)
  - Другие служебные базы данных

- **mbtiles/** - Тайлы карт
  - Формат: `.mbtiles`
  - Используется для хранения тайлов карт

### Документация (docs/)

Все файлы документации проекта собраны в одной папке для удобства навигации.

### Скрипты (scripts/)

Утилиты и скрипты миграции для управления данными и структурой базы данных.

## Управление путями

Все пути к файлам централизованно управляются через модуль `database/file_paths.py`:

```python
from database.file_paths import (
    get_avatar_path,
    get_project_path,
    get_file_path,
    get_database_path,
    get_mbtiles_path,
    ensure_directories
)
```

## Правила организации

1. **Все загруженные файлы** должны находиться в `uploads/` с подпапками по типам
2. **Все данные приложения** должны находиться в `data/` с подпапками по типам
3. **Вся документация** должна находиться в `docs/`
4. **Все скрипты** должны находиться в `scripts/`
5. **Используйте централизованные пути** из `file_paths.py` вместо хардкода путей

## Добавление новых типов файлов

При добавлении нового типа файлов:

1. Создайте подпапку в соответствующей директории (`uploads/` или `data/`)
2. Добавьте константу пути в `database/file_paths.py`
3. Добавьте функцию `get_{type}_path()` в `file_paths.py`
4. Обновите `ensure_directories()` для создания новой директории
5. Обновите эту документацию
