# Миграция метаданных HEC-RAS проектов

Этот документ описывает процесс миграции метаданных для существующих проектов HEC-RAS.

## Описание

Миграция извлекает метаданные из .db файлов для всех существующих проектов в базе данных. Метаданные включают:
- Параметры карты (maxzoom, centerx, centery, centerz)
- Границы (bottom, top, left, right)
- Информацию о проекте (project_name, plan_name, map_type)
- Легенду (legend_values, legend_rgba)

## Способы запуска миграции

### 1. Через скрипт командной строки

```bash
cd backend
python migrate_hecras_metadata.py
```

Скрипт обработает все проекты в базе данных и обновит метаданные для тех, у которых они отсутствуют или неполные.

### 2. Через API endpoint

Отправьте POST запрос на:
```
POST /api/hec-ras/migrate-metadata
```

Требуется аутентификация. Миграция будет выполнена только для проектов текущего пользователя.

**Пример запроса:**
```bash
curl -X POST "http://localhost:8000/api/hec-ras/migrate-metadata" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

**Пример ответа:**
```json
{
  "status": "completed",
  "total_projects": 5,
  "updated": 3,
  "skipped": 2,
  "errors": 0,
  "error_details": null
}
```

## Что делает миграция

1. Находит все проекты HEC-RAS в базе данных
2. Для каждого проекта:
   - Проверяет наличие .db файла
   - Если файл существует, извлекает метаданные через `extract_hecras_data()`
   - Обновляет запись в базе данных, если метаданные отсутствуют или неполные
3. Логирует результаты миграции

## Логика обновления

Проект будет обновлен, если:
- Метаданные полностью отсутствуют (`project_metadata` = NULL)
- Метаданные не содержат ключевых полей (например, `*_maxzoom`, `*_centerx`, `*_legend_values`, и т.д.)

Проект будет пропущен, если:
- Файл .db не найден
- Метаданные уже содержат все необходимые поля

## Логирование

Скрипт выводит подробную информацию о процессе миграции:
- Количество обработанных проектов
- Количество обновленных проектов
- Количество пропущенных проектов
- Количество ошибок
- Детали ошибок (если есть)

## Безопасность

- Миграция не удаляет существующие метаданные
- При обновлении существующие метаданные объединяются с новыми (новые имеют приоритет)
- Все изменения выполняются в транзакциях (rollback при ошибках)

## Примечания

- Убедитесь, что все .db файлы доступны по путям, указанным в `filepath`
- Миграция может занять некоторое время для большого количества проектов
- Рекомендуется сделать резервную копию базы данных перед миграцией
