# Визуализация HEC-RAS: Описание работы и используемые библиотеки

## Обзор

Визуализация HEC-RAS — это веб-приложение для отображения результатов гидродинамического моделирования HEC-RAS на интерактивной карте. Система использует архитектуру клиент-сервер, где фронтенд отображает данные на карте, а бэкенд предоставляет тайлы (tiles) и метаданные через REST API.

## Архитектура

### Фронтенд (React)

Основной компонент визуализации — `HecRasViewer.jsx`, который реализует следующие функции:
- Отображение карты с помощью Mapbox GL JS
- Загрузка и отображение тайлов HEC-RAS данных
- Временная анимация (time series animation)
- Интерактивная легенда с цветовой кодировкой
- Расчет и отображение площади затопления
- Графики изменения площади во времени

### Бэкенд (FastAPI)

Бэкенд предоставляет REST API для:
- Генерации тайлов из базы данных SQLite (HEC-RAS .db файлы)
- Предоставления метаданных (bounds, zoom levels, временные метки)
- Авторизации и управления проектами
- Публичного доступа через share_hash

## Основные библиотеки

### Фронтенд

#### 1. **mapbox-gl** (v3.12.0)
Основная библиотека для отображения интерактивных карт.

**Использование:**
- Создание и управление интерактивной картой
- Отображение растровых слоев (raster layers) с тайлами HEC-RAS
- Поддержка различных стилей карты (streets, satellite, satellite-streets)
- 3D рельеф (terrain) и небо (sky layer)
- Управление проекциями карты

**Ключевые особенности:**
```javascript
import mapboxgl from 'mapbox-gl';
import 'mapbox-gl/dist/mapbox-gl.css';

// Создание карты
const map = new mapboxgl.Map({
  container: mapContainer,
  style: 'mapbox://styles/mapbox/streets-v12',
  center: [82.6, 48.5],
  zoom: 6,
  pitch: 60,
  bearing: -17.6,
  projection: 'mercator'
});

// Добавление растрового источника тайлов
map.addSource('db-tiles', {
  type: 'raster',
  tiles: [`/api/map/tiles/${projectHash}/{z}/{x}/{y}.png`],
  tileSize: 256,
  bounds: mapMetadata.bounds,
  minzoom: mapMetadata.minzoom,
  maxzoom: mapMetadata.maxzoom
});

// Добавление растрового слоя
map.addLayer({
  id: 'db-tiles-layer',
  type: 'raster',
  source: 'db-tiles',
  paint: { 'raster-opacity': opacity }
});
```

## Принцип работы визуализации

### 1. Загрузка проекта

При открытии визуализатора HEC-RAS происходит следующее:

1. **Загрузка метаданных проекта:**
   - Запрос к `/api/map/metadata/{projectHash}` или `/api/map/metadata/shared/{shareHash}`
   - Получение bounds (границы), center (центр), minzoom, maxzoom, has_time

2. **Загрузка данных проекта:**
   - Запрос к `/api/hec-ras/{projectHash}` для получения metadata и original_filename
   - Определение префикса легенды из имени файла или метаданных

3. **Загрузка временных данных (если есть):**
   - Запрос к `/api/map/times/{projectHash}` или `/api/map/times/shared/{shareHash}`
   - Получение списка временных меток для анимации

4. **Загрузка легенды:**
   - Попытка загрузки из метаданных проекта
   - Fallback на `/api/uploads/{projectHash}/legend`
   - Отображение цветовой шкалы с диапазонами значений

### 2. Настройка карты

После загрузки метаданных настраивается карта:

1. **Инициализация Mapbox карты:**
   - Создание карты с выбранным стилем (streets, satellite, satellite-streets)
   - Установка начальной позиции (bounds или center + zoom)
   - Настройка pitch (наклон) и bearing (поворот) для 3D эффекта

2. **Добавление растрового источника тайлов:**
   - URL шаблон: `/api/map/tiles/{projectHash}/{z}/{x}/{y}.png` или `/api/map/tiles/shared/{shareHash}/{z}/{x}/{y}.png`
   - Поддержка временного параметра: `?time={timeValue}`
   - Настройка bounds, minzoom, maxzoom из метаданных

3. **Добавление растрового слоя:**
   - Слой типа 'raster' с источником тайлов
   - Настройка прозрачности (raster-opacity)

### 3. Загрузка тайлов

Mapbox GL JS автоматически запрашивает тайлы по мере необходимости:

1. **Определение необходимых тайлов:**
   - На основе текущего zoom level и видимой области карты
   - Расчет координат тайлов (z, x, y) в формате XYZ

2. **Запрос тайлов к серверу:**
   - GET запрос к `/api/map/tiles/{projectHash}/{z}/{x}/{y}.png`
   - Передача параметра time (если выбрано время)
   - Авторизация через Bearer token (для приватных проектов)

3. **Обработка тайлов на сервере:**
   - Открытие SQLite базы данных проекта
   - Определение схемы таблиц тайлов (autodetect)
   - Запрос тайла из таблицы с фильтрацией по time (если есть)
   - Декомпрессия PNG данных из базы
   - Возврат PNG изображения 256x256 пикселей

4. **Отображение тайлов:**
   - Mapbox GL JS автоматически отображает полученные тайлы
   - Кэширование тайлов в браузере
   - Плавная загрузка при перемещении и масштабировании карты

### 4. Временная анимация

Если проект содержит временные данные:

1. **Управление временем:**
   - Слайдер для выбора временной метки
   - Кнопки Play/Pause для автоматической анимации
   - Интервал обновления: 1 секунда

2. **Обновление тайлов:**
   - При изменении времени обновляется URL источника тайлов
   - Добавляется параметр `?time={currentTime}`
   - Mapbox автоматически запрашивает новые тайлы
   - Старые тайлы заменяются новыми

3. **Пересчет площади затопления:**
   - При изменении времени пересчитывается площадь затопления
   - Запрос к `/api/map/flood-area-time-series/{projectHash}`
   - Получение временного ряда площадей для графика

### 5. Расчет площади затопления

Площадь затопления рассчитывается на основе видимых тайлов:

1. **Получение границ видимой области:**
   - `map.getBounds()` — границы видимой области карты
   - `map.getZoom()` — текущий уровень масштаба

2. **Запрос к API:**
   - GET `/api/map/flood-area/{projectHash}?bounds={west},{south},{east},{north}&zoom={zoom}`
   - Передача параметра time (если выбрано)

3. **Отображение результата:**
   - Площадь в км² в модальном окне
   - График изменения площади во времени (если есть временные данные)

### 6. Легенда

Легенда отображается на основе метаданных проекта:

1. **Парсинг метаданных:**
   - Ключи вида `{prefix}_legend_values` и `{prefix}_legend_rgba`
   - Парсинг диапазонов значений ("0 - 5", "> 10", и т.д.)
   - Парсинг RGBA цветов (r, g, b, a для каждого диапазона)

2. **Отображение:**
   - HTML список с цветными квадратиками
   - Подписи диапазонов значений с единицами измерения
   - Автоматическое обновление при изменении проекта

### 7. Дополнительные возможности

1. **Управление стилем карты:**
   - Переключение между стилями Mapbox (streets, satellite, satellite-streets)
   - Сохранение предпочтений пользователя

2. **Управление проекцией:**
   - Поддержка различных проекций (mercator, globe, и т.д.)
   - Настройка предпочтений пользователя

3. **Управление картой:**
   - Zoom In/Out кнопки
   - Recenter кнопка (возврат к исходным границам)
   - Автоматический fitBounds при первой загрузке

4. **3D рельеф (опционально):**
   - Добавление terrain слоя из Mapbox DEM
   - Sky layer для атмосферного эффекта
   - Настройка exaggeration (преувеличение рельефа)

## Структура данных

### Формат тайлов

- **Формат:** PNG изображения 256x256 пикселей
- **Система координат:** XYZ tiles (Web Mercator, EPSG:3857)
- **Хранение:** SQLite база данных в сжатом виде (PNG)
- **Схема:** Автоматическое определение структуры таблиц

### Метаданные проекта

```json
{
  "bounds": [west, south, east, north],
  "center": [longitude, latitude],
  "minzoom": 0,
  "maxzoom": 18,
  "has_time": true
}
```

### Легенда

```json
{
  "mode": "normal",
  "legends": [{
    "title": "Plan01",
    "key_prefix": "Plan01",
    "classes": [
      {
        "label": "0 - 1",
        "hex": "#0000ff",
        "min": 0,
        "max": 1
      },
      {
        "label": "1 - 2",
        "hex": "#00ff00",
        "min": 1,
        "max": 2
      }
    ]
  }]
}
```

## Производительность и оптимизация

1. **Кэширование тайлов:**
   - Кэш на сервере для часто запрашиваемых тайлов
   - Кэш в браузере через HTTP заголовки Cache-Control

2. **Ленивая загрузка:**
   - Тайлы загружаются только для видимой области
   - Загрузка тайлов при перемещении и масштабировании

3. **Debounce для пересчета площади:**
   - Задержка 800ms перед пересчетом при перемещении карты
   - Предотвращение избыточных запросов

4. **Оптимизация запросов:**
   - Пакетная обработка запросов тайлов
   - Приоритизация видимых тайлов

## Безопасность

1. **Авторизация:**
   - Bearer token для приватных проектов
   - Проверка прав доступа на бэкенде

2. **Публичный доступ:**
   - Share hash для публичного доступа
   - Опциональная защита паролем
   - Отдельные эндпоинты для публичного доступа

3. **CORS:**
   - Настройка CORS заголовков для запросов Mapbox
   - Ограничение источников запросов

## Заключение

Визуализация HEC-RAS представляет собой комплексное решение для отображения результатов гидродинамического моделирования на интерактивной карте. Система использует современные веб-технологии (React, Mapbox GL JS, FastAPI) для обеспечения высокой производительности и удобства использования. Архитектура позволяет масштабировать систему и легко добавлять новые функции.
